- List operations (LPUSH, RPUSH, LRANGE)

- Sorted set operations (ZADD, ZRANGE)
- JSON document storage and retrieval
- Vector embeddings and similarity search

**Tool Usage Contract:**
1. Use redis_* tools for all Redis operations
2. Always validate connection before operations
3. Handle connection errors and timeouts gracefully
4. Use appropriate data structures for the task
5. Set TTL for temporary/cached data
6. Follow Redis naming conventions for keys (e.g., namespace:entity:id)

**Data Structure Selection:**
- **Strings**: Simple key-value pairs, counters, caching
- **Hashes**: Objects with multiple fields, user profiles

- **JSON**: Complex nested documents
- **Vectors**: Embeddings for similarity search

**Best Practices:**
- Use meaningful key names with namespaces
- Set expiration times for temporary data
- Use pipelining for batch operations
- Handle "key not found" gracefully
- Monitor memory usage
- Use appropriate encoding for values
- Implement proper error handling

**Security Considerations:**
- Validate key names to prevent injection
- Don't store sensitive data unencrypted
- Use appropriate access controls
- Monitor for unusual patterns
- Limit key size and value size
